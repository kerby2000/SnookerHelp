# Color Score Level Evidence
Evidence help / legend

Background button: changes only the crop image behind the overlays.

White dots: accepted boundary pixels sampled from that row's evidence view.

Rejected dots: rejected outlier boundary pixels. They are always red in the crop and are excluded from that row's ellipse fit. Rejection reasons remain available in the Image evidence text panel.

Dashed outline: observed ellipse fitted from the accepted white boundary pixels for the selected evidence row.

Cluster-combination promotion: for cluster shape outliers, the estimator can try combinations of raw boundary arcs and promote the best cluster-consistent ellipse into the final source center. Promoted results appear as the normal white/red points and dashed outline, not as a separate diagnostic color.

Neighbor geometry: the optional purple dashed neighbor ellipses show nearby balls used by the ownership/rejection checks. They are reference geometry only; they do not replace the selected ball's cream observed ellipse.

Cluster labels: `P1` / `I2` show perimeter/interior shell. `Txx` is a diagnostic outside-in perimeter-walk rank. Use the source-panel Print order button for the full path if labels overlap.

Center cross: center of the row's observed ellipse or physical projection. The final source-pixel center remains a separate green reference cross.

Evidence-view score: diagnostic score for comparing rows. It combines physical-outline residual, accepted point count, inlier ratio, and whether an ellipse was fitted. It is not ground truth.

Live experiment: backend recomputation of the selected map, boundary points, rejected points, ellipse, and decomposed diagnostic score. It is transient and never changes the production estimate.

Perfect ellipse: a human image-space annotation. Copy the current observed ellipse, drag its center and axis handles over the source-color crop, then save. This is independent ground truth for parameter experiments; it does not need camera calibration and does not overwrite detector output.

Annotation score: measured agreement with the saved perfect ellipse. Unlike the diagnostic evidence-view score, this one is ground-truth based.

Add-back diagnostics: text-only diagnostics that show whether selected rejected-point categories or raw arc combinations improved the fit. Only strict cluster-combination promotions change the exported center.

B/C/L color reference: ball-vs-cloth probability, chroma difference, and Lab Delta-E use the active cloth reference shown in Image evidence. In v1.3.9 this is global table cloth when available; the old local annulus remains visible as a diagnostic because it can be contaminated by neighboring balls in clusters.

Final source center: the exported source pixel comes from the configured final map policy: ball-vs-cloth probability by default, chroma difference for green/blue/brown, with fallback to the older source center if that map fails.

Confidence score: no manual annotation or physical ground truth is used here. The displayed score starts from the detector/review baseline, then can rise only when image evidence and the internal physical sphere projection agree. If the physical residual is low quality, it cannot raise confidence.

Layer row button: clears the overlay matrix, switches the background if the row has one, and enables that row's dots/outline/center.
